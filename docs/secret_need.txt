Docker images should be imutable ( same ) for every environment.
For that you cant bake ( put ) environment variable in image.
You can do that at docker run command or docker-compose command. But not at image build time,

example 1 -> docker run -e DB_HOST=localhost -e DB_USER=root myapp
example 2 -> docker run --env-file .env myapp
example 3 -> docker-compose.yml
            services:
                app:
                    image: myapp
                    environment:
                    - DB_HOST=${DB_HOST}
                    - DB_USER=${DB_USER
matab build karta huva docker-ignore ma .env honi chahiya. ya phir .env file exist hi na karti ho project ma.
